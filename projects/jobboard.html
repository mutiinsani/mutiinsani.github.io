LinkedIn Studentwerk Data

Automated pipeline + dashboard for working-student jobs in data & AI

Overview

This project collects LinkedIn job listings for working-student roles in data-related fields (Data Analyst, Data Scientist, AI/ML Engineer) in Germany, processes the data, and presents interactive visualizations in a Flask web app. It automates the end-to-end workflow: extraction → transformation → visualization → deployment.

What it does

  • Data Extraction — Scrapes job listings from a public website and saves raw CSVs in outputs/.
  • Data Transformation — Cleans and transforms raw CSVs, then writes processed files to processed/.
  • Data Visualization — Serves processed results through a Flask app. Using simple templates/ and static/.
  • Automation & CI — A GitHub Action triggers the pipeline (run main.py then etl-process.py on success).
  • Deployment — The Flask app is deployed on Render and rebuilds automatically on new commits.

Project Structure

Key files and folders:

  • main.py — scraper that extracts listings and writes raw CSVs to outputs/.
  • etl-process.py — ETL script that reads raw CSVs, cleans & transforms, and outputs to processed/.
  • app.py — Flask application that renders the job board page using templates and static assets.
  • templates/, static/ — frontend resources for the Flask app.
  • outputs/, processed/ — storage for raw and processed CSVs respectively.
  • flask-requirements.txt — Python dependencies for deployment.

How it works

1. Data extraction

The scraper (main.py) visits a public job listing page, gest jobs in the past 24 hour, parses job info, and writes raw CSV files to outputs/. Each run collects new listings and appends or stores timestamped CSVs for traceability.

2. Data transformation

etl-process.py reads raw CSVs, handles missing values, normalizes fields (titles, locations, company names), and creates derived features (e.g., role_category, role_category_order). The cleaned dataset is saved into processed/.

3. Data visualization

The Flask app (app.py) loads processed CSVs and shows:

  • List of relevant jobs only
  • Job search box

Automation (GitHub Actions)

A GitHub Action workflow runs main.py once a day. If the scraper succeeds, it triggers etl-process.py. This ensures the processed dataset is always rebuilt after successful scrapes.

Deployment (Render)

The Flask app is deployed on Render. The service is connected to the GitHub repository and configured to rebuild automatically on commits to selected paths.
